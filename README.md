# Columbia CT-6 Boiler Dosing Controller

A comprehensive, open-source boiler water treatment system designed to replicate and extend the features of commercial industrial controllers (like the Lakewood Instruments Model 1575e and Walchem WBL400/410 Series). 

This repository contains both the embedded C++ firmware for the ESP32 micro-controller and a Dockerized backend for time-series data storage and visualization.

## Repository Structure

- **[`firmware/esp32_boiler_controller/`](firmware/esp32_boiler_controller/)**: The core embedded C++ application. Built using PlatformIO, it runs on an ESP32 micro-controller and handles all hardware IO, RTOS task management, fuzzy logic control, and network telemetry.
- **[`backend/`](backend/)**: A fully containerized stack using Docker Compose. It features a Node.js Express API server, TimescaleDB for telemetry storage, and Grafana for dashboards.
- **`docs/`**: Technical design documents, Standard Operating Procedures (SOPs), and architecture details (e.g., Fuzzy Logic Control, Coprocessor Communication).
- **`controller_manuals/`**: Reference manuals for commercial controllers that informed this project's feature set.

## Main Components & Features

### Embedded Firmware (ESP32)
* **Chemical Dosing Control:** Manages up to 3 NEMA 17 Stepper motors (via A4988 drivers) for precise, volumetric chemical dosing (e.g., H2SO3, NaOH, Amine).
* **Conductivity & Temperature Sensing:** Interfaces with an Atlas Scientific EZO-EC circuit via UART and a PT1000 RTD temperature probe via an Adafruit MAX31865 module (SPI). 
* **Automated Blowdown:** Controls a blowdown valve dynamically based on continuous conductivity readings.
* **Local HMI & Interfaces:** 
  * 20x4 I2C LCD Display for immediate system status.
  * WS2812 RGB LEDs for quick visual alarm/status indication.
  * Embedded **Async Web Server** as a modern local HMI for on-site configuration and monitoring.
* **Resilience & Control:** Integrates FreeRTOS for task scheduling (Control, Measurement, Display, Logging), local SD Card logging as a fallback, and NVS for persistent configurations. Supports an RS-485 "Coprocessor Link" for splitting measurement and control logic across separate panels.

### Backend Stack
* **API Server:** An Express.js REST API that ingests telemetry (sensor readings, events, alarms) from the ESP32.
* **MQTT Gateway:** Natively subscribes to MQTT topics (`device/+/metrics`, `device/+/alarm`) and pushes data directly into the database.
* **Data Storage:** TimescaleDB (PostgreSQL) optimized for handling heavy time-series data generated by sensor measurements.
* **Visualization:** Pre-configured Grafana dashboards utilizing the TimescaleDB datasource to visualize real-time water conditions, pump activity, and alarm states.

## Getting Started

### 1. Backend Setup (Server/Raspberry Pi)

1. Ensure [Docker](https://docs.docker.com/get-docker/) and Docker Compose are installed.
2. Navigate to the backend directory:
   ```bash
   cd backend
   ```
3. Copy the example environment variables and update passwords/keys:
   ```bash
   cp .env.example .env
   ```
4. Start the stack:
   ```bash
   docker compose up -d
   ```
5. Access the services:
   - API: `http://<ip>:3000`
   - Grafana: `http://<ip>:3001` (Login with `admin` and your configured password)

*For more details, see the [Backend README](backend/README.md).*

### 2. Firmware Setup (ESP32)

1. Open this repository in an IDE with the **PlatformIO** extension installed (e.g., VS Code or Cursor).
2. Configure your specific environment parameters (WiFi SSID, API Key, Backend IP, etc.) via the `config.h` file or through the local Web UI.
3. Build and upload the firmware to your ESP32 device using the appropriate environment (e.g., `esp32dev` or `step1_base`):
   ```bash
   pio run -e esp32dev -t upload
   ```
4. Once booted, the ESP32 will connect to WiFi, synchronize time via NTP, start the control loop, and begin posting telemetry to the backend API or MQTT broker.

*For test programs and calibration routines, see the [Test Programs README](firmware/esp32_boiler_controller/test_programs/README.md).*

* **Infrastructure:** Docker, Docker Compose, Grafana.
